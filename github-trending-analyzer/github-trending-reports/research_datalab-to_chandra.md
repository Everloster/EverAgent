# datalab-to/chandra

> OCR model that handles complex tables, forms, handwriting with full layout.

## 项目概述

Chandra OCR 2 是 Datalab 团队开源的文档智能识别模型，可将图片和 PDF 转换为保留布局信息的结构化 HTML/Markdown/JSON，支持 90+ 语言、数学公式、复杂表格、手写体和表单重建。在 olmocr 基准测试中综合得分 85.9（满分 100），超越所有开源模型。与传统 OCR 的核心区别在于"文档理解"而非"文本识别"——不仅识别文字，还保留原始布局，理解表格和数学公式，输出可直接使用的数据结构。

## 技术分析

### 技术栈

- **Python**：推理框架，支持 HuggingFace 本地推理和 vLLM 服务端
- **vLLM 优化**：生产级推理优化，单卡 H100 80GB 可达 1.44 pages/s
- **两种推理模式**：本地（HuggingFace）+ 远程（vLLM Server）

### 核心功能

- **布局保留输出**：HTML/Markdown/JSON 多种格式，保持原始文档结构
- **90+ 语言支持**：包括中文（88.7% 准确率）、阿拉伯语、日文、印地语等
- **数学公式识别**：复杂数学表达式、手写公式均可处理
- **表格重建**：保持表格结构，输出可编辑数据
- **手写体识别**：支持手写表单、笔记识别
- **图像提取**：自动提取文档中的图片并添加标题

## 总结评价

### 优势

- 多语言领先：43 语言平均 77.8%，超越 Gemini 2.5 Flash
- 布局保留：输出可直接使用，非纯文本
- 开源免费：Apache 2.0 代码，OpenRAIL-M 模型（非商业用途免费）
- 多种输出格式：HTML/Markdown/JSON 灵活选择

### 劣势

- 非商业用途限制：模型使用 OpenRAIL-M 协议，需商业授权才能用于竞品服务
- 复杂表格（合并单元格）识别仍有误差
- 吞吐量相对较低（1.44 pages/s）

### 适用场景

- 文档数字化：历史档案、图书馆数字化
- RAG 知识库：PDF 文档的结构化提取
- 表单处理：银行、保险、政府表单自动处理
- 学术研究：论文、专利文档的快速提取
- 多语言文档处理：跨国企业文档本地化
