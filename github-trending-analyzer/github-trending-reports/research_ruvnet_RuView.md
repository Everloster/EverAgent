# ruvnet/RuView

> π RuView: WiFi DensePose turns commodity WiFi signals into real-time human pose estimation, vital sign monitoring, and presence detection — all without a single pixel of video.

## 项目概述

RuView 是一款利用商品 WiFi 设备（如普通路由器）的 CSI（Channel State Information）信号，通过深度学习实现无摄像头人体姿态估计、生命体征监测和存在感知的开源项目。其核心创新在于"用看不见的信号做视觉级感知"——不需要摄像头、不需要可穿戴设备，仅凭 WiFi 信号即可实现人员计数、跌倒检测、呼吸监测等功能。v0.5.0 版本融合 mmWave 毫米波传感器，ESP32-S3 固件发布，在隐私保护感知赛道处于领先地位。

## 技术分析

### 技术栈

- **Rust 主导（5.1M 行代码）**：高性能边缘 AI 推理，ESP32 固件核心
- **Python**：深度学习模型训练与数据处理
- **mmWave 融合**：v0.5.0 新增毫米波传感器融合，提升精度
- **ESP32-S3 CSI 固件**：商品路由器/IoT 设备即可部署

### 架构设计

信号采集 → 特征提取 → 神经网络推理 → 姿态/生命体征输出：

```mermaid
flowchart LR
    A[WiFi CSI 信号] --> B[ESP32-S3 固件采集]
    B --> C[Rust 高性能预处理]
    C --> D[神经网络推理引擎]
    D --> E[姿态估计 / 生命体征]
    E --> F[隐私保护感知输出]
```

### 核心功能

- **无摄像头姿态估计**：通过 WiFi 信号实现人体骨骼检测
- **生命体征监测**：呼吸频率、心率检测（无需可穿戴设备）
- **存在感知**：人员计数、区域入侵检测
- **mmWave 融合**：v0.5.0 新增毫米波增强，提升复杂环境精度
- **隐私保护**：无视频采集，适用于医疗、养老等敏感场景

## 总结评价

### 优势

- 隐私保护：无摄像头，符合 GDPR、医疗隐私法规
- 高性能：Rust 主导，边缘设备即可运行
- 多场景：智能家居、医疗监护、老年陪护、安防
- 开源：MIT 协议，硬件无关

### 劣势

- 复杂环境（多房间、金属干扰）精度下降
- 毫米波版本硬件成本较高
- 多人场景下的个体区分仍有挑战

### 适用场景

- 智能家居：无摄像头的人员感知
- 医疗监护：老人跌倒检测、呼吸监测
- 养老陪护：非侵入式生命体征追踪
- 安防监控：隐私敏感区域的入侵检测
