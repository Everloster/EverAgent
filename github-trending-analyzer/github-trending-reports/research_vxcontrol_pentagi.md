# vxcontrol/pentagi

> PentAGI: Fully autonomous AI Agents system capable of performing complex penetration testing tasks - An innovative security testing platform leveraging artificial intelligence to automate sophisticated penetration testing workflows in sandboxed Docker environments.

## 项目概述

PentAGI是一个突破性的自主AI渗透测试系统，由VXControl开发，专门为安全测试和漏洞评估自动化而设计。该项目融合了先进的多智能体架构、知识图谱系统和20+专业安全工具的集成，在隔离的Docker沙箱环境中自主执行复杂的渗透测试任务。截至2026年3月，项目已获得13,111个Stars，代表了AI在网络安全领域应用的重要突破。项目强调了AI驱动的自动化如何能够显著提升安全测试的效率和覆盖范围。

## 技术分析

### 技术栈

PentAGI采用了现代的微服务和前后端分离架构，技术栈设计精良：

**后端系统**:
- **核心语言**: Go，高性能并发处理，适合构建可靠的分布式系统
- **API框架**: GraphQL API，提供灵活的数据查询和操作接口
- **数据库**: PostgreSQL，配备pgvector扩展支持向量搜索和语义查询
- **知识图谱**: Neo4j数据库，用于存储和管理复杂的关系数据
- **内存存储**: Redis用于缓存和会话管理
- **任务队列**: 支持分布式任务调度和异步处理

**前端系统**:
- **框架**: React，采用TypeScript确保类型安全
- **状态管理**: 支持实时数据更新的响应式设计

**LLM生态**:
- **多供应商支持**: 10+个LLM提供商集成
  - OpenAI GPT系列
  - Anthropic Claude系列
  - Google Gemini
  - AWS Bedrock
  - 本地部署: Ollama、Llama 2等开源模型
  - 其他支持: xAI Grok、国内LLM模型

**基础设施**:
- **容器化**: Docker Compose部署，提供完整的微服务编排
- **监控与可观测性**:
  - Prometheus用于指标收集
  - Grafana用于实时监控仪表板
  - Langfuse用于LLM可观测性和调试
  - Jaeger用于分布式追踪
  - 结构化日志系统

**安全工具集成**:
- 20+专业渗透测试工具
  - 网络扫描: nmap、masscan
  - 漏洞检测: Nessus、Qualys
  - Web应用测试: Burp Suite、OWASP ZAP、sqlmap
  - 密码破解: hashcat、john
  - Payload生成与执行: metasploit、Empire
  - 其他工具: aircrack-ng、volatility等

### 架构设计

PentAGI的架构设计体现了多智能体系统的最佳实践：

**多智能体系统**:
- **研究员智能体（Researcher Agent）**: 专注于开源情报(OSINT)和信息收集，执行网站扫描、DNS枚举、公开数据库查询等任务
- **开发员智能体（Developer Agent）**: 根据特定防御机制编写或修改脚本和Payload，生成绕过WAF、IDS等防护的自定义攻击代码
- **执行员智能体（Executor Agent）**: 与命令行接口交互，实际运行安全工具，执行攻击链

**知识管理系统**:
- **Graphiti集成**: 基于Neo4j的时间知识图谱，追踪信息的时序关系和演变
- **长期记忆**: 利用向量嵌入和语义搜索，存储和检索过往的渗透测试经验
- **语义理解**: 能够理解不同信息之间的语义关系，构建完整的攻击图

**监督与控制机制**:
- **自动导师干预**: 当执行模式出现问题时，系统自动提出纠正建议
- **执行监控**: 检测无限循环和死锁，推荐替代策略
- **智能任务规划**: 将复杂目标分解为可执行的原子任务
- **工具调用限制**: 防止资源耗尽和无限重试

**沙箱隔离**:
- 所有操作在Docker容器中执行，完全隔离
- 支持多容器协同，每个容器可配置不同的工具集和操作系统
- 网络隔离选项，支持气隙(air-gapped)部署

### 核心功能

1. **自主任务执行**: 系统根据高层目标自动规划和执行测试步骤，无需人工干预

2. **漏洞挖掘与利用**: 不仅发现漏洞，还能自动生成利用代码和Payload

3. **详细报告生成**: 输出包含漏洞描述、利用步骤、补救建议的专业渗透测试报告

4. **实时监控与可视化**: 通过Grafana仪表板实时监控AI智能体的执行过程

5. **LLM可观测性**: 通过Langfuse追踪每个LLM调用，进行调试和优化

6. **浏览器集成**: 容器化的浏览器用于Web应用测试和JavaScript执行

7. **多搜索源集成**: 支持Tavily、DuckDuckGo、Perplexity、Google Custom Search等

8. **持久化存储**: 将测试结果、漏洞数据、报告存储在PostgreSQL中，支持查询和分析

## 总结评价

### 优势

- **创新架构**: 多智能体系统结合知识图谱，充分利用AI和传统安全工具的优势
- **自主能力**: 真正的自主执行能力，减少人工干预，提高效率
- **完整生态**: 集成了安全工具、监控、可观测性，形成完整的解决方案
- **隔离安全**: Docker沙箱确保测试的安全性和可控性
- **可扩展性**: 支持多个LLM供应商，易于扩展新的安全工具
- **企业级**: 提供了企业部署所需的监控、日志、报告功能

### 劣势

- **法律与合规**: 渗透测试涉及敏感的法律问题，用户必须确保获得授权
- **成熟度**: 作为新兴项目，在特定攻击场景下的可靠性仍需验证
- **配置复杂**: Docker Compose部署和多个外部系统的配置可能对初学者有难度
- **LLM局限**: LLM的推理能力在某些复杂的漏洞利用上可能不足
- **成本**: 部署完整堆栈需要显著的计算资源和LLM API调用成本
- **误报风险**: 可能生成虚假漏洞或不可靠的利用步骤
- **学习曲线**: 需要理解渗透测试、LLM、知识图谱等多个领域

### 适用场景

**高度适用**:
1. **安全研究机构**: 研究AI在安全领域的应用
2. **大型企业**: 进行定期的合规性渗透测试
3. **安全咨询公司**: 提高测试规模和效率
4. **DevSecOps团队**: 集成到CI/CD流程中的自动化安全测试
5. **Bug赏金平台**: 发现和利用安全漏洞

**需谨慎使用**:
1. **小型组织**: 配置和维护成本较高
2. **关键基础设施**: 需要额外的安全验证和合规评估
3. **跨境测试**: 需要明确的法律授权和合规性确认
4. **实时防御**: 不适合用于攻击溯源和实时响应
