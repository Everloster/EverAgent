# czlonkowski/n8n-mcp

> A Model Context Protocol (MCP) server that bridges n8n's workflow automation platform with AI models, providing comprehensive access to n8n node documentation, properties, and operations.

## 项目概述

n8n-mcp 是一个创新的项目，通过实现 Model Context Protocol 标准，将 n8n 强大的工作流自动化能力与 AI 模型（包括 Claude、Windsurf、Cursor 等）连接起来。该项目让 AI 助手能够深入理解 n8n 的 1,084 个工作流节点，并能自动化生成、验证和优化完整的工作流，显著降低复杂自动化任务的学习成本和开发周期。

## 实现技术与扩展

### MCP 协议的核心原理

Model Context Protocol (MCP) 是由 Anthropic 定义的开放标准协议，用于 AI 模型与外部工具、数据源的通信。在 n8n-mcp 中的应用：

```
Claude AI (或其他支持 MCP 的客户端)
    │
    ├─ 请求：告诉我如何创建一个邮件通知工作流
    │
    ▼
n8n-mcp Server
    │
    ├─ 搜索相关节点：GMail、Slack、邮件等
    ├─ 检索文档：各节点的参数和用法
    ├─ 构建工作流框架
    ├─ 验证配置有效性
    │
    ▼
Claude AI
    │
    └─ 响应：这是生成的工作流 JSON...
```

MCP 的优势：
- **标准化通信**: 统一的请求-响应格式
- **工具发现**: 客户端可自动发现 Server 提供的能力
- **安全沙箱**: 限制 AI 模型对系统的直接访问

### n8n 节点的分类体系

1,084 个节点按功能分类：

**集成类节点 (300+ 个)**
- CRM: Salesforce, HubSpot, Pipedrive
- 电商: Shopify, WooCommerce, Stripe
- 通信: Gmail, Slack, Teams, Telegram
- 云存储: AWS S3, Google Drive, Dropbox

**数据处理节点 (150+ 个)**
- 数据库: PostgreSQL, MySQL, MongoDB
- HTTP/API: REST, GraphQL
- 数据转换: JavaScript, Python, SQL

**自动化节点 (200+ 个)**
- 流程控制: 条件、循环、并行
- 定时任务: Cron, Webhooks
- 错误处理: Try-Catch, 重试逻辑

**AI/ML 节点 (100+ 个)**
- 大模型: OpenAI, Claude, Gemini
- 向量数据库: Pinecone, Weaviate
- 智能处理: 文本分析, 图像识别

### 工作流生成的 AI 算法

n8n-mcp 的工作流生成涉及几个关键步骤：

**第 1 步: 需求理解**
- 用户用自然语言描述任务
- AI 提取关键信息：触发条件、处理逻辑、输出操作

**第 2 步: 节点选择**
- 搜索符合条件的 n8n 节点
- 评估多个备选方案
- 选择最佳组合（考虑成本、兼容性、性能）

**第 3 步: 工作流框架生成**
- 构造节点连接图
- 配置数据流
- 添加错误处理

**第 4 步: 验证与优化**
- 检查所有节点参数的有效性
- 验证数据类型匹配
- 优化执行效率

**第 5 步: 用户审查与迭代**
- 展示生成的工作流
- 用户可要求修改
- 循环优化直到满意

### 性能与成本考量

**执行性能指标**

- n8n 单个工作流执行时间：0.5-30 秒（取决于节点数和外部 API 响应）
- MCP 通信开销：50-200ms（额外的 RTT）
- 总体影响：对用户而言基本透明

**成本优化**

相比 Zapier 的一个巨大优势是成本模型：
- **Zapier**: 按"任务"（task）计费，一个 5 步工作流 = 5 个 task 消耗
- **n8n**: 按"执行"计费，一个 50 步工作流 = 1 个执行消耗
- **成本差异**: 处理 10,000 条数据，Zapier 需要 500,000 个任务，n8n 仅需 10,000 个执行

**示例计算**：
```
场景：每月处理 50,000 条客户数据，进行 CRM 同步

Zapier:
- 50 步工作流 × 50,000 执行 = 2,500,000 任务
- 成本：$500-2000/月（取决于套餐）

n8n-mcp (自托管):
- 50 步工作流 × 50,000 执行 = 50,000 执行
- 成本：$50 服务器成本 + 人工成本

节省比例：90% 以上成本削减
```

## 深度探讨：AI 驱动工作流的颠覆性

### 传统工作流设计 vs AI 驱动设计

**传统工作流设计流程（Zapier/Make 模式）**

```
业务需求文档
    ↓ (理解)
确定集成应用
    ↓ (搜索，通常需要多次尝试)
学习平台界面
    ↓ (需要 2-4 小时学习曲线)
配置各个节点
    ↓ (手动拖拽，配置参数)
测试工作流
    ↓ (需要多次迭代)
部署上线
总耗时: 4-8 小时
错误率: 15-20%（配置错误、遗漏步骤）
```

**AI 驱动工作流设计流程（n8n-mcp 模式）**

```
用自然语言描述需求
    ↓ (输入)
Claude 通过 n8n-mcp 理解能力
    ↓ (使用 MCP 查询节点库和文档)
自动选择合适节点和配置
    ↓ (AI 决策，基于 1,084 个节点和 2,709 个模板)
生成完整的工作流 JSON
    ↓ (输出可直接使用的配置)
自动验证所有配置
    ↓ (检查参数有效性)
部署上线
总耗时: 2-5 分钟
错误率: 2-5%（AI 验证机制）
```

**时间节省统计**

对于一个复杂的 10 步工作流：
- 传统方式: 6 小时（包括学习、配置、测试、修复）
- n8n-mcp 方式: 15 分钟（描述 → 生成 → 验证 → 上线）
- 效率提升: 24 倍

### n8n-mcp 的技术创新点

**1. 知识库的构建**

n8n-mcp 包含的知识库数据：
- 1,084 个节点的完整文档
- 每个节点的 99% 属性覆盖
- 87% 的文档完整性
- 2,709 个工作流模板
- 265 个 AI 工具优化变体

这个知识库的规模使得 Claude 能够精确理解 n8n 的能力，而不是猜测或产生幻觉。

**2. 验证机制的完善**

不同于一般的 LLM 生成内容，n8n-mcp 的验证机制确保了生成内容的正确性：

```
生成工作流 JSON
    ↓
验证第 1 层: 语法检查（JSON 格式）
    ↓
验证第 2 层: 节点存在性（确保节点在 n8n 中存在）
    ↓
验证第 3 层: 参数有效性（数据类型、值范围检查）
    ↓
验证第 4 层: 数据流连接（输出类型与输入类型匹配）
    ↓
验证通过 ✓
```

这个多层验证机制大幅降低了错误率。

**3. 工作流推理的智能化**

Claude 在 n8n-mcp 的辅助下，可以：
- 理解业务逻辑并推断需要的节点组合
- 根据数据类型自动配置转换节点
- 预判可能的错误并添加错误处理
- 提出性能优化建议

示例：用户说"将 CSV 文件中的客户数据导入 CRM"
- Claude 推断: 需要文件读取 → CSV 解析 → 数据验证 → CRM API 调用
- 自动选择: Read File 节点 → Parse CSV 节点 → JavaScript 验证 → Salesforce 节点
- 添加错误处理: Try-Catch 分支 + 邮件通知

### 与现有自动化工具的融合可能性

**与 Zapier 的互操作性**

理论上可以构建"Zapier-mcp"，让 Claude 也能驱动 Zapier 工作流。但面临挑战：
- Zapier 的文档和 API 不如 n8n 开放
- n8n-mcp 能充分利用 n8n 的开源特性
- 竞争关系导致合作困难

**与 Make 的整合**

Make 有类似的自动化能力，理论上也可以基于 MCP 开发。优势：
- Make 在低代码工作流方面的设计更成熟
- 强大的数据转换功能
- 挑战：同样的闭源限制

**与业界 RPA 工具的结合**

RPA（机器人流程自动化）工具如 UiPath、Blue Prism 可以通过 MCP 与 Claude 集成：
- 高度复杂的自动化任务（GUI 操作）由 RPA 完成
- 简单的数据处理由 n8n 完成
- AI 作为总指挥协调 RPA 和 n8n
