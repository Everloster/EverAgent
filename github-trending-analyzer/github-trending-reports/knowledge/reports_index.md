## 报告规范

每个报告包含以下7个章节：
1. 项目概述（定位与核心价值）
2. 基本信息（Stars/Forks/语言/协议等）
3. 技术分析（技术栈/架构/核心功能）
4. 社区活跃度（贡献者分析/Issue-PR/动态）
5. 发展趋势（版本演进/Roadmap/反馈）
6. 竞品对比（≥2个竞品对比表）
7. 总结评价（优势/劣势/适用场景）

## 数据来源

- GitHub API（仓库信息、贡献者、Issue、PR、Release）
- 网络搜索（项目概述、竞品信息、社区反馈）
- 官方文档和 README
