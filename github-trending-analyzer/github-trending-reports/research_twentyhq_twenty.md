# twentyhq/twenty

> Building a modern alternative to Salesforce, powered by the community.

## 项目概述

Twenty 是一个开源 CRM 平台，目标非常明确：成为现代化、社区驱动、可自托管的 Salesforce 替代方案。与本次日榜上大多数 agent/AI 项目不同，它属于“业务软件基础设施”类型，这也是它在榜单里相当醒目的原因。

从 Twenty 官网与 GitHub 数据看，它已经不再是“一个新奇开源 CRM”，而是逐步形成产品、社区、路线图和商业叙事兼备的成熟项目。对于关注开源商业软件的人来说，这个仓库的热度说明市场仍然在寻找可控、可定制、非闭源锁定的 CRM 方案。

## 技术分析

### 技术栈

- **主语言**：TypeScript
- **前后端形态**：React + NestJS 风格的现代 Web 应用栈
- **数据层**：官方页面和 topic 指向 PostgreSQL、GraphQL、Web
- **仓库形态**：Monorepo

### 架构设计

Twenty 本质上是“业务系统平台”，而不是单功能工具。其架构重点在：

- **可配置对象与字段**
- **工作流自动化**
- **API / Webhooks 集成**
- **社区驱动路线图**

官网把产品定位为“customer data operating system”，这说明它希望承载的不只是传统联系人管理，而是更广义的客户数据与流程协作。

### 核心功能

- CRM 核心对象管理
- 字段与对象自定义
- 工作流自动化
- API 与 Webhook 集成
- 社区驱动 roadmap

## 发展趋势

### 版本演进

- **起点**：面向开源 CRM 赛道
- **中期**：扩展为更灵活的 customer data 平台
- **当前**：明显强化社区、自动化和开放路线图叙事

### Roadmap

可预期的演进方向包括：

- 更强对象建模与自动化
- 更多第三方系统集成
- 更丰富的权限与企业特性
- 更成熟的自托管运维体验

### 社区反馈

Twenty 的吸引力在于它同时覆盖了两个痛点：

- 闭源 CRM 成本高、锁定强
- 传统开源 CRM 往往界面与开发体验落后

但它也面对现实挑战：CRM 是深水区产品，功能边界大，实施与迁移成本高，社区热度并不自动等于企业落地成功。

## 总结评价

### 优势

- 赛道清晰，商业价值明确
- 社区规模和贡献者数量可观
- 技术栈现代，开发者友好
- 开源与可自托管叙事强

### 劣势

- CRM 领域实现深度要求极高
- 许可信息在 GitHub 元数据与官网表述之间需要用户自行确认
- 企业真实迁移与实施成本仍然不低

### 适用场景

- 需要开源 CRM 的创业公司和成长型团队
- 重视可定制和自托管的组织
- 希望在现代 Web 技术栈上扩展 CRM 的开发团队
