## 🌍 Step 1 | 背景与动机（WHY）

### 扩散模型的架构演进问题

2022 年，扩散模型在图像生成质量上已全面超越 GAN，但存在一个悬而未决的问题：

```
架构选择：
  Pixel-space DM (DDPM/ADM)：CNN U-Net 骨架
  Latent DM (LDM/SD)：CNN U-Net 骨架 + 预训练 VAE

两者共同点：都是基于 CNN U-Net 的设计
  → 这是继承自原始 DDPM 的历史包袱
  → 并非经过系统评估后的最优选择
```

### 为什么 Transformer 可能是更好的选择？

三个层面的考量：

```
1. Scaling 潜力
   - Transformer 在 NLP 和视觉领域已证明具有出色的 scaling 特性
   - U-Net 在超过一定规模后 scaling 收益递减
   - GPT-3/4、ViT 的成功暗示 Transformer 是 scaling 的最优架构之一

2. 架构统一
   - Transformer 已统一了 NLP（GPT、T5）→ 视觉分类（ViT）→ 多模态（CLIP、GPT-4V）
   - 若扩散模型也迁移到 Transformer → 全模态架构统一
   - "A model that can process all modalities"

3. 充沛的工程经验
   - Transformer 的训练技巧（Tricks）已被深度研究
   - 硬件效率（FlashAttention 等）成熟
   - 可直接复用 LLM 训练的 infrastructure
```

### LDM 的局限性

LDM (Stable Diffusion) 证明了潜空间扩散的效率优势，但其 UNet 骨架并未改变：
- UNet 在大模型下的 efficient attention 方法不够成熟
- UNet 的 scaling 特性未被系统研究
- 与 LLM 的 infrastructure 无法共享

## 💡 Step 2 | 技术方案（WHAT & HOW）

### 核心贡献

1. **DiT (Diffusion Transformer)**：首个纯 Transformer 骨架的扩散模型，替代 U-Net
2. **Scaling Laws 验证**：证明 DiT 遵循 compute-law，模型规模↑ = 质量↑
3. **自适应机制**：AdaLN/AdaLN-Zero 比 IN/MLP Conditioning 更高效
4. **SOTA 生成质量**：DiT-XL/2 在 256×256 ImageNet 上 FID 2.27，超越 BigGAN、ADM、LDM

### DiT 架构详解

#### 整体设计

```
输入：潜空间图像 z ∈ R^{h×w×c}（来自预训练 VAE，c=4 for LDM）

Patchify：将图像转换为 patch 序列
  z → patches: 每个 patch = 2×2×c → flattened 4c vector
  patch size = 2, patch 数 = (h/2)×(w/2)
  → 序列长度 = (h/2)×(w/2)

  例如 256×256 图像（VAE 下采样 8×）：
    h = w = 32（潜空间尺寸）
    patch size = 2
    → 16×16 = 256 个 patches
    → 序列长度 256，与 ViT-B/16 相同

线性嵌入：每个 patch → D 维 embedding
位置编码：标准 ViT 位置编码（可学习或 RoPE）

然后通过 N 个 DiT block 处理序列：
  DiT Block = LayerNorm + Cross-Attention（条件）+ MLP
  条件输入：time embedding + class embedding（AdaLN）
```

#### 自适应机制的选择（关键消融实验）

论文系统比较了 4 种条件注入方式：

```
(a) In-Context Conditioning (in_channels)
    → 将条件（time + class）作为额外 tokens 拼接到输入序列前
    → 类似于 Flamingo 的 in-context learning

(b) Cross-Attention
    → 序列前加 [T] [C] token
    → DiT block 内: attention([patch_tokens], [T,C])
    → 额外 attention 层，引入参数和计算开销

(c) AdaIN (Adaptive Instance Normalization)
    → 将条件注入到 IN 的 scale 和 shift 参数
    → 来自 StyleGAN 的设计

(d) AdaLN (Adaptive Layer Normalization)
    → 将条件注入到 LN 的 scale 和 shift 参数
    → γ = linear(t) + linear(c), β = linear(t) + linear(c)

(e) AdaLN-Zero ⭐（最终选择）
    → 在 AdaLN 基础上：
       - 初始化：γ=0, β=0（除以恒等函数）
       - 将 MLP 输出乘以 0 再加回（zero 初始化 trick）
       - 最后添加残差连接
    → 优势：训练初期模型等价于无条件的 DiT
           渐进式引入条件信号，训练更稳定
```

AdaLN-Zero 的核心思想来自 ResNet 的 skip connection 初始化：zero-γ/β 让残差路径主导训练过程。

#### DiT Block 设计

```
标准 DiT Block（基于 ViT）：

Input: x (B, N, D)  [B=batch, N=patches, D=hidden_dim]
  ↓
LayerNorm
  ↓
├── 自注意力：x → W_Q, W_K, W_V → Attention(x, x)
├── 残差连接 + LayerNorm
├── AdaLN 调制：γ(c), β(c) from condition embedding c
│   γ(c) = linear(c),  β(c) = linear(c)
├── MLP: linear(d) → GELU → linear(d)
├── 残差连接
│
└── Output

条件信号处理：
  time embedding t ∈ R^d → 线性 → γ_t, β_t
  class embedding c ∈ R^d → 线性 → γ_c, β_c
  γ = γ_t + γ_c,  β = β_t + β_c
```

#### 模型配置（与 ViT 命名一致）

| 配置 | 参数量 | Gflops | Patch Size | 对应 ViT |
|------|--------|--------|-----------|---------|
| DiT-S | 39M | 167G | 2 | ViT-S |
| DiT-B | 345M | 1185G | 2 | ViT-B |
| DiT-L | 657M | 2291G | 2 | ViT-L |
| DiT-XL | 675M | 2519G | 2 | Viهير-XL |

特别注意：DiT-B/2（patch size 2）vs DiT-XL/2，参数相近但计算量差很多。

### Classifier-Free Guidance（推理时）

论文使用 classifier-free guidance (CFG) 提升条件生成质量：

```
条件分数估计（CFG）：
  ε_θ(z_t, c) = (1+w)·ε_θ(z_t, c) - w·ε_θ(z_t, ∅)

w = guidance scale（无分类器引导强度）
c = 条件（如 class label）
∅ = 空条件（null condition）

更高的 w → 更好的条件对齐，但多样性下降
DiT-XL/2 在 w=1.6 时 FID 最优
```

## 🔍 Step 3 | 理论支撑与论证

### Scaling Laws 的验证

这是论文最重要的理论贡献——证明 DiT 遵循 scaling laws：

```
实验设置：
  - 训练：400K steps，batch 256，LDM 潜空间（f=4 VAE）
  - 测试：FID vs compute（不同规模的 DiT）

核心发现（Fig. 3）：
  1. 固定 compute，模型规模↑ → FID ↓
  2. 固定模型规模，steps↑ → FID ↓（更多迭代更好）
  3. 最优策略：先 scale 模型，再增加采样步数

关键：DiT 的 scaling 曲线与语言模型的 scaling laws 高度相似
  → 说明 Transformer 架构具有跨领域的通用 scaling 特性
  → 扩展 DiT 比扩展 UNet 更有效率
```

### 为什么 AdaLN-Zero 比其他 Conditioning 机制更好？

```
理论分析：

1. 渐进式条件引入：
   - 训练初期：γ=β=0 → DiT = 无条件模型
   - 随训练进行：条件信号逐渐引入
   - 不会因强条件信号导致训练不稳定

2. 参数效率：
   - AdaIN/AdaLN：2D → 只需 γ, β（2×D 参数）
   - Cross-Attention：额外 attention 层（4×D² 参数）
   - In-Context：增加序列长度（额外计算）

3. 残差路径保持：
   - AdaLN 不破坏残差连接
   - 训练初期：skip connection = identity
   - 随训练进行：残差 + AdaLN 联合学习
```

### 位置编码的选择

论文比较了：
- 标准可学习位置编码（与 ViT 相同）
- 傅里叶位置编码（FiLM 条件）

两种效果相近，但最终选择了与 ViT 一致的可学习编码。

## 📊 Step 4 | 实验评估

### 主要结果：Class-Conditioned ImageNet 256×256

| 方法 | FID ↓ | IS ↑ | Precision ↑ | Recall ↑ | 参数量 |
|------|-------|------|------------|---------|--------|
| BigGAN-deep | 6.95 | 203.6 | 0.87 | 0.28 | 160M |
| ADM | 10.94 | 101.0 | 0.68 | 0.63 | 554M |
| ADM-U (BigGAN conv) | 4.59 | 186.7 | 0.82 | 0.52 | 554M |
| LDM-8 (f=8, 条件生成) | 15.61 | 79.0 | - | - | 400M |
| LDM-4 (f=4) | 3.60 | 247.7 | 0.84 | 0.57 | 400M |
| **DiT-XL/2 (无 CFG)** | 9.95 | 121.3 | - | - | 675M |
| **DiT-XL/2 (w=1.0)** | 5.20 | 165.6 | 0.78 | 0.57 | 675M |
| **DiT-XL/2 (w=1.6)** | **2.27** | **278.4** | **0.83** | **0.57** | 675M |

DiT-XL/2 + CFG w=1.6 达到 FID 2.27，**超越所有此前方法**（包括 BigGAN-deep、ADM）。

### Scaling 曲线分析

Fig. 4 的关键发现：
- DiT-L/2 和 DiT-XL/2 在 400K 步后仍未饱和
- 更大模型 + 更多步数 = 更好的 FID
- **DiT-XL/2 能在 35 个采样步数下达到 ADM 800 步的同等质量**

### 消融实验关键发现

```
1. Conditioning 机制对比（400K 步，DiT-B/2）：
   In-Context: FID 24.0
   Cross-Attention: FID 21.1
   AdaIN: FID 15.9
   AdaLN: FID 9.1
   AdaLN-Zero: FID 8.6 ✅

2. Patch Size 影响（DiT-XL）：
   Patch Size 2: 2519 Gflops, FID 9.6
   Patch Size 4: 624 Gflops, FID 11.0
   Patch Size 8: 169 Gflops, FID 13.9
   → Patch Size 2 最优（以计算换质量）

3. 模型深度 vs 宽度：
   - 增加深度（D=18 vs D=36）比增加宽度（D=1024 vs D=512）更有效
   - 推荐：Deep + Narrow > Shallow + Wide
```

### 采样效率突破

这是 DiT 最重要的 practical impact：

```
ADM（像素空间 U-Net）：需要 250-1000 步采样
LDM-4（潜空间 U-Net）：需要 50-100 步采样
DiT-XL/2（潜空间 Transformer）：只需要 **25-35 步** 即达到 SOTA 质量

原因：Transformer 的 scaling 让模型更强
     → 可以在更少步数内完成高质量去噪
     → 推理速度提升 ~2-3 倍
```

                                                      Unified Architecture

"A Transformer to rule them all" 的最后一块拼图：

### Bitter Lesson 视角

DiT 是 Bitter Lesson 的完美验证：

```
Bitter Lesson 的预言：
  → "通用方法利用计算能力，最终总是赢"

DiT 的验证：
  → "Transformer 替换 U-Net，随计算增加持续改善"
  → UNet 在某规模后饱和，Transformer 没有
  → 计算资源增加时，Transformer 的优势更加明显

核心洞察：
  放弃手工设计的 U-Net 归纳偏置（局部性、权重共享）
  → 拥抱通用的 Transformer 架构
  → 让 scaling 接管优化工作
```

## 🤔 Step 6 | 个人理解

### 最重要的洞察

DiT 最重要的贡献不是"Transformer 能做扩散生成"（这是自然的尝试），而是**证明了 Transformer 在扩散模型中同样遵循 scaling laws**：

```
关键发现：
  - DiT-B/2 vs DiT-XL/2 的 scaling 曲线与 GPT-2 vs GPT-3 高度相似
  - 计算量翻倍 → 质量线性改善（对数 FID vs log compute 近似线性）
  - 这意味着扩散模型的 scaling 存在清晰的可预测路径

实践意义：
  → 可以提前预估"需要多少 compute 达到目标 FID"
  → 工程投资有据可依
  → Scaling 是扩散模型质量提升的主要杠杆
```

### 为什么 AdaLN-Zero 的 zero-initialization 有效？

这是一个深刻的技术细节，与 ResNet 的 skip connection initialization 一脉相承：

```
标准初始化问题：
  - 若 γ, β 随机初始化，训练初期条件信号会破坏预训练行为
  - 这在深层网络中尤为严重

ResNet 的洞察（2015）：
  - 残差分支初始化为 0 → 训练初期等价于浅网络
  - 随训练进行逐步引入新能力

AdaLN-Zero 的同样洞察：
  - γ=0, β=0 的 LN → 恒等函数
  - MLP 输出乘以 0 → 零贡献
  - 训练初期 = 无条件 DiT
  - 随训练 → 逐步引入条件调制

这种"渐进式解锁"的初始化策略是深度学习的重要 trick，
在 diffusion 领域的成功应用展现了跨领域的知识迁移。
```

### 用一个类比解释

```
DiT 对扩散模型的意义，就像 ViT 对视觉分类的意义：

ViT 之前：CNN 统治视觉，局部特征 = 归纳偏置
ViT 之后：Transformer 证明"局部性不是必要的"
          大数据 + 全局 attention = 超越 CNN

DiT 之前：U-Net 统治扩散，局部算子 = 归纳偏置
DiT 之后：Transformer 证明"局部性不是必要的"
          大模型 + 全局 attention = 超越 U-Net

共同模式：
  "放弃手工归纳偏置 → 拥抱通用架构 → 让 scaling 接管"
  → Bitter Lesson 的又一次验证
```

### 局限与开放问题

1. **位置编码的时序问题**：DiT 处理图像 patch（空间结构），视频生成需要时序位置编码（尚未解决）
2. **计算量仍然很高**：DiT-XL/2 仍需 2500 Gflops/采样步，高于 LDM 的 UNet
3. **潜空间依赖**：DiT 依赖 LDM 的潜空间（VAE），不是端到端的像素级生成
4. **无条件生成**：DiT 的 scaling 主要在条件生成上，无条件生成的质量提升有限
