# Large-Scale Data Filtering（大规模数据过滤）

> 在 billion-to-trillion 级别数据集上系统性移除噪声、重复和低质量内容的技术

## 一句话定义

大规模数据过滤是在海量原始数据（CommonCrawl、互联网文本、图文对）上，通过 URL 去重、语义质量评分、语言识别、毒性检测等多级过滤手段，从根本上提升训练数据质量，使大模型能够在更大规模上学习到更纯净信号的技术工程。

## 核心挑战

```
Billion 级别数据无法人工审核：
  - LAION-5B：58.5 亿图文对
  - RefinedWeb：5T tokens
  - 人工逐条审核 = 不可能

必须使用自动化方法：
  → 但简单规则（长度、字符）效果有限
  → 需要语义级判断

核心矛盾：
  严格过滤 → 数据量锐减
  宽松过滤 → 噪声过多
```

## 关键技术

### 1. URL / 文档去重

```
目的：移除重复抓取的同一内容

URL 去重：
  - 规范化 URL（去除 tracking 参数）
  - 精确匹配去重
  - 通常去除 30-50% 的 URL

文档级去重（MinHash + LSH）：
  - 将文档分成 n-gram shingles
  - 计算 MinHash 签名
  - Jaccard > 0.8 的文档对视为重复
  - 保留最长/最高质量的

效果（RefinedWeb）：
  - 额外去除 5-10% 文档
  - 大幅提升训练效率
```

### 2. 语言识别

```
目的：识别并保留目标语言文本

工具：FastText 轻量语言分类器
  - 训练快，推理快
  - 支持 176 种语言

阈值策略：
  - 英文置信度 > 0.65 保留
  - 低于阈值 = 目标语言之外

影响：
  CommonCrawl 约 50% 是英文
  语言过滤后约 20-25% 保留
```

### 3. 质量过滤

```
统计特征过滤：
  - 文档长度（<200 或 >50000 字符）
  - 特殊字符比例
  - 句子长度分布
  - 标点密度

语言模型打分：
  - 训练小型 LM（~1M 参数）
  - 计算文档 perplexity
  - 过高 = 噪声语言（乱码、模板）
  - 过低 = 重复/高度模板化

结构分析：
  - 检测是否为导航页（无实质内容）
  - 检测是否为登录页/弹窗页面
  - 保留有完整文章结构的文档
```

### 4. 毒性过滤

```
目的：移除有害内容

关键词过滤（保守）：
  - 明显违规关键词
  - 高精度但不完整

二分类模型：
  - Perspective API 类似模型
  - 毒性分类（暴力、仇恨、性虐待）
  - 阈值经过人工标注校准

设计原则：
  宁可少过滤，不要误过滤
  → 避免文化偏差
  → 避免误删边缘内容
```

### 5. CLIP 图文配对过滤（LAION 专用）

```
用于：图文对数据集（LAION）

方法：
  1. 使用 CLIP 计算图文相似度
  2. similarity > 阈值（如 0.32）保留
  3. 低于阈值 = 图文不相关

阈值选择：
  - 0.32 = 质量-规模平衡点
  - 低于 0.32 = 大量图文不匹配
  - 高于 0.32 = 规模损失过大
```

## 数据集对比

| 指标 | The Pile | RefinedWeb | LAION-5B |
|------|---------|-----------|---------|
| 规模 | 825GB | 4.6TB | 2.7TB（图） |
| Tokens | ~300B | ~5T | 58.5 亿图文对 |
| 来源 | 22 数据源混合 | CommonCrawl | CommonCrawl |
| 去重 | 粗粒度 | 细粒度（URL+文档） | URL+CLIP |
| 语言过滤 | 轻微 | 系统性 | 系统性 |
| 质量过滤 | 基础 | 多级渐进 | CLIP 评分 |

## 与其他概念的关系

```
大规模预训练数据构建：
  CommonCrawl ──→ URL 去重 ──→ 文本提取 ──→ 语言过滤
                                          ↓
                          质量过滤 ──→ 毒性过滤 ──→ RefinedWeb
                          CLIP 评分 ──→ LAION-5B
                                        ↓
                              高质量预训练数据
                                        ↓
                              Falcon LLM / Stable Diffusion
```

## 方法论意义

```
RefinedWeb 提出的 Progressive Filtering：
  1. 不同维度问题 → 不同工具检测
  2. 逐步精细化，而非一步到位
  3. 宁可保留边缘样本，不要误删

核心洞察：
  "让模型（LM perplexity）来评价数据质量"
  → 符合 Bitter Lesson 的"通用方法 > 人类规则"
```
