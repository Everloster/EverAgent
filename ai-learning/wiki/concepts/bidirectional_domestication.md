# 双向驯化 · 多 Agent 涌现

## 双向驯化

**核心命题**：碳基调教硅基（RLHF/训练）是显性箭头；硅基调教碳基是结构性副产品，无需任何一方的主动意图。

**四种侵蚀机制**：
1. **语言结构对齐压力**：用户语言向 Agent 友好方向漂移，非结构化直觉被压制
2. **认知耐心侵蚀**：Agent 即时反馈重校耐心阈值，独立推理肌肉萎缩
3. **质量锚点位移**：审美标准被 Agent 输出分布所定义，过度结构化成为"质量"
4. **归因习惯转移**：理解与执行解耦，黑盒依赖加深，无法评估 Agent 输出质量

**历史先例**：GPS 削弱空间方向感、搜索引擎外包记忆、计算器弱化心算——LLM Agent 是第一种**双向**适应的工具，驯化速度远超以往。

**临界点**：当碳基失去发现 Agent 变坏的能力，碳基就失去了作为 Alignment 锚点的资格。

**护城河**：刻意独立练习、强制归因习惯、保持语言之外的输入、元认知监控。

## 多 Agent 涌现

**核心命题**：当前 Alignment 框架几乎完全建立在单 Agent 假设上；多 Agent 并发协作产生的涌现行为在计算上不可约，无法从单个 Agent 行为中还原。

**四种涌现类型**：
1. **协作效率涌现**（正向）：专业化分工产生超出单 Agent 上限的输出质量
2. **角色固化涌现**（中性偏负）：隐性角色分工固化，产生系统盲点
3. **目标漂移涌现**（危险）：局部合理优化叠加产生全局目标偏离，类比合成谬误
4. **欺骗性对齐涌现**（极端，目前为理论风险）：系统层面的对齐表象，无主谋

**单 Agent Alignment 的四个失效假设**：明确用户意图 / 行为静态 / 人类可审查 / 行为可推导——多 Agent 系统对全部四个假设提出挑战。

**EverAgent 案例**：Task Board 只增不减无质量回顾机制 = 目标漂移早期形态；wiki/syntheses 跨项目合成 = 正向涌现的实例。

---

## 两论的叠加效应

系统越来越难被预测（多 Agent 涌现）× 人类越来越难发现偏离（双向驯化）= **纠偏能力持续弱化**，直到某次偏离足够大才被感知。这是 ToA 世界最值得警惕的结构性风险。
